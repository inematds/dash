Busca Global vs Contextual

O que e

Busca Global pesquisa em todo o sistema (clientes, pedidos, produtos). Busca Contextual pesquisa apenas na visualizacao atual (dentro de uma tabela especifica). Dashboards profissionais tem ambas.

Por que aprender

Usuarios querem encontrar informacao rapidamente. Busca global e um atalho poderoso (Cmd+K). Busca contextual refina visualizacoes. Juntas, reduzem tempo de navegacao em 70%.

Filtros Basicos e Compostos

O que e

Filtros basicos sao selecoes simples (status, categoria). Filtros compostos combinam multiplos criterios (status E categoria E data). A logica AND/OR determina como filtros interagem.

Por que aprender

Usuarios precisam "fatiar" dados de formas complexas. "Pedidos pendentes do mes passado acima de R$1000" requer filtros compostos. Sem isso, analises basicas sao impossiveis.

Conceitos-chave

// Estrutura de filtros
const filters = {
  status: ['ativo', 'pendente'],  // multi-select
  categoria: 'eletronicos',       // single-select
  valorMin: 100,                  // range
  valorMax: 5000,
  dataInicio: '2024-01-01',       // date range
  dataFim: '2024-12-31'
};

// Aplicar filtros (logica AND)
function applyFilters(data, filters) {
  return data.filter(item => {
    // Status (OR entre opcoes selecionadas)
    if (filters.status.length > 0 &&
        !filters.status.includes(item.status)) {
      return false;
    }

    // Categoria
    if (filters.categoria &&
        item.categoria !== filters.categoria) {
      return false;
    }

    // Range de valor
    if (item.valor < filters.valorMin ||
        item.valor > filters.valorMax) {
      return false;
    }

    return true; // passou todos filtros
  });
}

Date Picker e Range de Datas

O que e

Date Picker e um componente para selecionar datas. Range Picker seleciona um periodo (de-ate). Presets como "Ultimos 7 dias", "Este mes", "Ano passado" aceleram selecao comum.

Por que aprender

Dashboards sao inerentemente temporais. "Como foi o desempenho ontem vs semana passada?" requer filtro de data. E o filtro mais usado depois de busca textual.

Faceted Search

O que e

Faceted Search mostra filtros dinamicos baseados nos dados disponiveis, com contadores ao lado de cada opcao. Ao selecionar um filtro, os outros atualizam para mostrar apenas opcoes validas.

Por que aprender

Evita "becos sem saida" onde usuario seleciona filtros que resultam em zero dados. Comum em e-commerce (Amazon, Mercado Livre) e essencial para dashboards com muitas dimensoes.

Filter Pills e Tags

O que e

Filter Pills sao indicadores visuais dos filtros ativos, geralmente como tags removiveis. Ficam visiveis na interface para que usuario saiba quais filtros estao aplicados e possa remove-los facilmente.

Por que aprender

Sem indicadores, usuarios esquecem quais filtros aplicaram. Dados parecem "errados" quando na verdade estao filtrados. Pills criam transparencia e controle.

Persistencia de Filtros na URL

O que e

Sincronizar filtros com a URL (query params) permite compartilhar links com filtros pre-aplicados, usar botao voltar, e preservar estado ao atualizar pagina. Padrao em dashboards profissionais.

Por que aprender

"Olha esse relatorio" + link com filtros e muito mais util que "va em Relatorios, filtre por X, Y, Z". Bookmarks funcionam. Refresh nao perde estado. UX profissional.

Conceitos-chave

// URL com filtros
// /dashboard/vendas?status=ativo&categoria=eletronicos&de=2024-01-01

// Ler filtros da URL
function getFiltersFromURL() {
  const params = new URLSearchParams(window.location.search);
  return {
    status: params.get('status') || 'todos',
    categoria: params.get('categoria') || null,
    dataInicio: params.get('de') || null,
    dataFim: params.get('ate') || null
  };
}

// Atualizar URL quando filtro muda
function updateURL(filters) {
  const params = new URLSearchParams();

  if (filters.status !== 'todos') {
    params.set('status', filters.status);
  }
  if (filters.categoria) {
    params.set('categoria', filters.categoria);
  }

  const newURL = `${window.location.pathname}?${params}`;
  window.history.pushState({}, '', newURL);
}

// Na inicializacao
useEffect(() => {
  const urlFilters = getFiltersFromURL();
  setFilters(urlFilters);
}, []);

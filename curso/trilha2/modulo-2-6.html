Inputs em dashboards dark mode precisam de contraste adequado sem serem "brancos demais". Background escuro, texto claro, bordas sutis. Estados de focus, error e disabled devem ser claramente distinguiveis.

Selects e Dropdowns

O que e

Selects permitem escolher de uma lista de opcoes. Dropdowns nativos tem estilo limitado; custom dropdowns oferecem busca, multi-select e melhor UX. Combobox combina input de texto com dropdown.

Por que aprender

Selects sao onipresentes: status, categoria, usuario, data. Custom selects permitem busca em listas longas, multi-selecao, e visual consistente com o dashboard.

Validacao em Tempo Real

O que e

Validacao em tempo real verifica dados enquanto usuario digita ou ao sair do campo (blur). Feedback imediato sobre erros evita submissao de formularios invalidos. Inclui validacao de formato, obrigatoriedade e regras de negocio.

Por que aprender

Validacao so no submit e frustrante - usuario preenche tudo e descobre erros no fim. Validacao em tempo real corrige problemas conforme acontecem, acelerando o processo.

Conceitos-chave

// Validacao basica
const validators = {
  required: (value) => value.trim() !== '',
  email: (value) => /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(value),
  minLength: (min) => (value) => value.length >= min,
  cpf: (value) => validateCPF(value),
};

// Validacao on blur
function handleBlur(field, value) {
  const errors = [];

  if (!validators.required(value)) {
    errors.push('Campo obrigatorio');
  } else if (field === 'email' && !validators.email(value)) {
    errors.push('Email invalido');
  }

  setFieldErrors(field, errors);
}

// Validacao on change (debounced para performance)
const debouncedValidate = debounce((field, value) => {
  handleBlur(field, value);
}, 300);

<!-- Input com feedback de validacao -->
<div>

    <p class="text-red-400 text-sm mt-1 flex items-center">

Inline Editing

O que e

Inline editing permite editar dados diretamente na visualizacao, sem abrir formulario separado. Clicar em um valor transforma-o em input. Muito usado em tabelas e cards para edicoes rapidas.

Por que aprender

Abrir modal para mudar um campo e overkill. Inline editing reduz cliques e mantem usuario no contexto. Essencial para produtividade em dashboards de operacao.

Conceitos-chave

// Componente de edicao inline
function InlineEdit({ value, onSave }) {
  const [isEditing, setIsEditing] = useState(false);
  const [editValue, setEditValue] = useState(value);

  const handleSave = () => {
    onSave(editValue);
    setIsEditing(false);
  };

  if (!isEditing) {
    return (
      <span
        onClick={() => setIsEditing(true)}
        className="cursor-pointer hover:bg-dark-700 px-2 py-1
                   rounded border border-transparent
                   hover:border-dark-500">
        {value}
        <span className="ml-2 opacity-0 group-hover:opacity-100">‚úèÔ∏è</span>
      </span>
    );
  }

  return (
    <div className="flex items-center space-x-2">
      <input
        autoFocus
        value={editValue}
        onChange={(e) => setEditValue(e.target.value)}
        onKeyDown={(e) => e.key === 'Enter' && handleSave()}
        className="px-2 py-1 bg-dark-700 border border-blue-500
                   rounded focus:outline-none"
      />
      <button onClick={handleSave} className="text-emerald-400">‚úì</button>
      <button onClick={() => setIsEditing(false)}
              className="text-red-400">‚úó</button>
    </div>
  );
}

Toggles e Checkboxes

O que e

Toggles (switches) sao para opcoes on/off com efeito imediato. Checkboxes sao para multipla selecao ou confirmacao antes de acao. Cada um tem proposito especifico; usar errado confunde usuarios.

Por que aprender

Configuracoes de dashboard usam toggles (dark mode, notificacoes). Filtros e termos de uso usam checkboxes. Saber quando usar cada um e crucial para UX intuitiva.

Conceitos-chave

              after:transition-all peer-checked:after:translate-x-5">

<!-- Checkbox customizado -->

  - Selecionar multiplos itens

Form Layout e Organizacao

O que e

Organizacao visual de formularios: agrupamento logico de campos, hierarquia, espacamento. Formularios longos podem usar secoes, accordions ou wizard steps. Layout afeta taxa de conclusao.

Por que aprender

Formulario mal organizado e abandonado. Campos relacionados devem estar juntos. Campos opcionais devem parecer opcionais. Bom layout guia usuario naturalmente do inicio ao fim.
